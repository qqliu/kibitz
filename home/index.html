Recommenders

Kibitz offers two different ways to build your recommenders and four different types of recommenders.

Creation Options

Ready-Built

Upload a table to Datahub with column names: "title", "description", and "image". Make the primary key of your table be "title". Kibitz will give you the option of either downloading a ZIP file containing your recommender to host at your own domain or hosting your recommender at kibitz.csail.mit.edu/YOUR_TABLE_NAME.

Customize

Customize your recommender using custom table columns. Your table must still contain a primary key. If you choose to use custom table columns, you will need to code your own recommender using our, easy to learn, Kibitz API.

Recommender Types

Mix, match, or use all four!

User-Based

Users create profiles and rate items. Recommended items will be matched to users who share similar interests.

Item-Based

Upload a file of item similarities or designate columns as similarity measures. Users will be recommended items that are similar to rated items.

Overall Ratings

Provide a column of average ratings. Highest rated items will be recommended to all users.

Random

Recommend to users completely random items from your items table.

Kibitz API

Code your recommender using our API!

Required Functions

You must first download supporting files from Kibitz Github Repo before using the following API. The following functions must be included at the top of your file. Use the client key you were emailed in place of CLIENT_KEY. This snippet sample is in Javascript. Go here for samples in other languages.

var transport = new Thrift.Transport("http://kibitz.csail.mit.edu:9888/kibitz/");
var protocol = new Thrift.Protocol(transport);
var client = new kibitz.RecommenderServiceClient(protocol);

transport.open();
client.createNewIndividualServer(CLIENT_KEY);
client.initiateModel(CLIENT_KEY, TABLE_NAME, DATAHUB_USERNAME, DATAHUB_PASSWORD, DATAHUB_DATABASE_NAME);

Other Functions

list<Item>makeRecommendation(CLIENT_KEY, long userId, long numRecs, list<string> displayColumns): Takes your client key, the user id of the user you want to make recommendations for, the number of recommendations you want to make, and a list of strings of columns you want to display on your website. Outputs a list of Item objects as recommended items.

list<Item> makeItemBasedRecommendations(CLIENT_KEY, long userId, long numRecs, list<string> displayColumns): Takes the client key, the user id of the user you want to make item-based recommendations for, the number of recommendations you want to make, and a list of strings of columns you want to display on your website. Outputs a list of Item objects as recommended items.

list<Item> getPageItems(CLIENT_KEY, long page, long numPerPage, list<string> displayColumns): Takes client key, the page number, the number of items per page, and the names of the columns that you want to display. Returns a list of Item objects to be displayed.

long getItemCount(CLIENT_KEY): Takes the client key and returns the total number of items in your items table.

void recordRatings(CLIENT_KEY, long userId, long itemId, long rating): Takes the client key, the user id of the user you want to record ratings for, the item id of the item that the user rated, and the rating and records it into the corresponding ratings table in Datahub.

void deleteRatings(CLIENT_KEY, long userId, long itemId): Takes client key, the user id of the user you want to delete ratings from, and the item id that you want to delete the rating for.

string createNewUser(CLIENT_KEY, string username, string email, string password, bool iskibitzuser): Takes the client key, the desired username of the new user to your website, the desired password of the user, and False. Returns whether the username already exists.

bool checkLogin(CLIENT_KEY, string username, string password, bool iskibitzuser): Takes your client key, the username of the user wishing to log in, the password of the user wishing to log in, and False. Returns whether the user has succesfully logged in.

long retrieveUserId(CLIENT_KEY, string username, string password): Takes your client key, the username of the user you want to retrieve the id of, and the password of the user. Returns the id of the user or -1 if the user does not exist.

list<Item> getUserRatedItems(CLIENT_KEY, long userId, list<string> displayColumns): Takes your client key, the user id of the user you want to retrieve rated items for, and a list of column names you want to display information about the item. Returns a list of Item objects that represent items the user has rated.

list<Item> getSearchItems(CLIENT_KEY, string query, list<string> displayColumns): Takes your client key, the search query your user searched, and the list of names of columns you want to display for each item. Returns a list of Item objects that match your search query.

list<Item> makeOverallRatingBasedOrRandomRecommendation(CLIENT_KEY, string ratingColumnName, long numRecs, list<string> displayColumns): Takes your client key, the column name of the column containing the ratings for each item, the number of recommendations you want to make, and a list of column names of info you want to display for each item. Return a list of Item objects that were chosen either using overall ratings or randomly.

list<Item> getItemsFromPrimaryKeys(CLIENT_KEY, string primaryKey, list<string> itemKeys, 4:list<string> displayColumns): Takes your client key, the the column name of the column that holds the primary keys in your table, a list of primary keys of items you want to return, and a list of column names you want to use to display your items. Returns a list of Item objects for the primary keys you queried.

About

Kibitz began as a research project in Haystack, a lab in MIT CSAIL. The purpose of the project is two-fold:
  1. Study the end-to-end application building process of non/inexperienced coders.
  2. Look into whether recommendations may be made for seemingly unrelated items.
